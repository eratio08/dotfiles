# Sources

## Canonical Source Policy

Use the latest stable Amber documentation as the canonical source for language behavior, syntax, CLI usage, and stdlib guidance.
Use the `amber` and `amber-docs` repositories as supplemental sources for realistic examples and to locate authoritative files.

## Primary Sources

Site: `https://amber-lang.com/`
Docs root: `https://docs.amber-lang.com/`
Docs repo: `https://github.com/amber-lang/amber-docs`
Compiler repo: `https://github.com/amber-lang/amber`

## High-Value Documentation Areas

Getting started and overview pages for language positioning and CLI workflow.
Basic syntax pages for variables, types, functions, conditions, loops, commands, and importing.
Testing pages for Amber test blocks.
Stable stdlib documentation pages for module APIs.

## Supplemental Example Sources

Use `amber-docs` by-example pages for larger worked examples when they align with stable docs.
Use Amber's own `.ab` files in the `amber` repository as realistic examples of command usage, imports, and script structure.
Treat repo examples as supplemental, not canonical, when version drift appears.

## Research Basis For This Skill

The skill content was based on the Amber homepage, the documentation site, the `amber-docs` repository, and the `amber` compiler repository.
The skill intentionally keeps only the minimum Amber rules in `SKILL.md` and pushes larger examples and extended explanations into `references/`.
